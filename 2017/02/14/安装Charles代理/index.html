破解(推荐购买正版)

  • 将 Charles.app 拖至 应用程序 文件夹
  • 复制 charles.jar 至 /Applications/Charles.app/Contents/Java/ 不清楚上面路径的可以通过（Finder—应用程序—Charles—显示包内容-Contents-Java）进入
  • 使用 SN.txt 中的序列码即可

手机通过Charles代理

以下内容基于Mac 4.0.2版本, iphone 5s

Mac抓取本地http请求

Charles安装完毕就可以抓取电脑上采用http协议的封包了，将Charles设为电脑端的网络代理即可。
harles菜单栏 -> Proxy -> Mac OS X Proxy, 勾选中

Mac代理手机并抓取手机http请求

  • 打开代理: harles菜单栏 -> Proxy -> Proxy Settings
  • 开启一个本地代理端口: Proxies列 -> HTTP Proxy -> Port 中填入8888(也可以是其他的，注意不要和本地进程冲突), 勾选Enable transparent HTTP proxying
  • 手机连接电脑: 设置 -> 无线局域网 -> 选择所连接wifi详情 -> HTTP 代理 -> 手动
    • 服务器填本地的ip地址，请注意自己的ip地址要固定
    • 端口填写第二步操作中的8888端口
  • 设置完成后打开任意网络请求，在Charles这边还需要做一个允许手机请求的操作, 点击Allow

Mac抓取手机HTTPS请求

Charles菜单栏 -> Help —> SSL Proxying —> Install Charles Root Certificate 输入系统账号和密码，就可以在KeyChain（钥匙串）中找到该证书。
解锁系统根证书 -> 搜索找到Charles Proxy Custom Root Certificate (XXXXX) -> 打开 ->选择信任 -> 使用此证书时：“始终信任”

此时则可以抓取到电脑端的https请求

Mac代理手机并抓取手机https请求

  • 手机安装证书: 在手机连上Charles代理的情况下，使用Safari打开http://charlesproxy.com/getssl, 会弹出一个证书让你安装。

    ! 不连接Charles的情况不会安装证书，会打开一个页面
    ! 注意此时安装的证书只能用于当前连接的电脑, 如果连接别的Mac还需要用同样的方法安装一遍
    ! 注意你要代理的APP可能有限制, 也许需要本地安装APP的证书才能抓取HTTPS的包

  • 配置Charles允许抓取HTTPS请求

    • 方法1: 在请求列表中右键链接，弹出菜单中选择 Enable SSL Proxying
    • 方法2： Charles菜单栏 -> Proxy -> SSL Proxying Settings -> SSL Proxying面板 -> 勾选Enable SSL Proxying -> 在Locations里添加add一个Host和port, 如(.xxx.com, 443), 或者可以直接 (, 443)

443端口一般用于HTTPS协议的传输端口,wiki

进行域名映射(link)

Charles菜单栏 -> Tools -> Map Remote… -> Enable Map Remote -> Add -> 添加完成后OK
前面From的是指请求来源，后面To的指想要转发的地址, 可以是本地址

本地HOST

  • 可以在在本地 /etc/hosts 中配置如 : 127.0.0.1 dev.xxx.com
  • 如自己的项目是在localhost:8000端口，则可以通过修改host的方式将地址改为 dev.xxx.com:8000同样可以访问
